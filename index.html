Project Scope

Literature Survey

The rapid advancement of Artificial Intelligence (AI), particularly through Generative AI (GenAI) and Large Language Models (LLMs) like GPT-4, has significantly impacted education by enabling adaptive and personalized learning experiences. Traditional education often relies on static, one-size-fits-all assessments that fail to address individual learning needs. In contrast, AI-powered systems offer dynamic content generation, real-time feedback, and adaptability based on student performance, which enhances both engagement and learning outcomes.

In Software Engineering, conventional assessments cannot keep pace with students' evolving understanding of complex concepts like OOP and SDLC. GPT-4 can generate context-aware, adaptive questions tailored to a learner’s proficiency, thereby improving conceptual understanding and problem-solving skills.

In English language learning, AI overcomes the limitations of static quizzes by offering personalized feedback, adaptive difficulty, and gamified learning experiences. This fosters higher engagement and retention in grammar, vocabulary, and comprehension skills.

Programming education also faces challenges in scaling question generation to diverse learning levels. Existing models often produce semantically weak content. By incorporating Retrieval-Augmented Generation (RAG) and semantic ontologies, the proposed system can produce contextually accurate and skill-appropriate programming problems.

Similarly, Mathematics education benefits from AI systems that dynamically adjust question complexity based on real-time learner performance. However, challenges remain in maintaining mathematical precision and curricular alignment. The integration of LangChain and LangGraph in the proposed system supports structured, accurate, and personalized question generation. Together, these advancements promise a transformative shift from rigid educational models to intelligent, scalable, and student-centered learning ecosystems.

Research Gap

Following areas are the research gaps found in most of the recent researches.

User Personalization

There are no widely adopted AI-driven quiz systems in Sri Lanka that dynamically tailor content based on individual student learning profiles. Each learner has unique strengths and weaknesses, yet most systems fail to generate personalized questions that adapt to a user’s learning journey.

Real-time Adaptability

Current quiz systems lack the capability to instantly adapt based on a student's past performance. There is no real-time feedback mechanism that intelligently regenerates questions from previously misunderstood topics, limiting the reinforcement of weak areas.

Difficulty Assessment

There is no automated mechanism in existing systems to classify users into difficulty tiers based on their performance. Adaptive difficulty levels—easy, medium, or hard—are not assigned dynamically, which results in mismatched question difficulty and reduced learning efficiency.

User Engagement

Traditional quiz platforms often lack visually appealing and interactive user interfaces. Without modern design principles and gamified feedback, these platforms struggle to maintain user motivation and long-term engagement.

SLIIT Module Relevant

No current system ensures that AI-generated questions are aligned specifically with SLIIT’s curriculum. As a result, learners receive content that may be academically irrelevant or misaligned with their institutional learning outcomes.

Research Problem & Solution

Proposed Problem

How can AI-driven question generation systems be developed to create contextually relevant and responsive questions that adapt to varying student proficiency levels across different domains?
What strategies can be employed to integrate AI-driven question generation systems with personalized learning techniques, ensuring that the system dynamically adjusts question difficulty based on real-time student performance and feedback?
How can Retrieval-Augmented Generation (RAG), and other advanced AI methods be applied to improve the contextual relevance, coherence, and pedagogical soundness of the generated questions?


Product Demonstration - Solution

Proposed Solution

The proposed system uses GPT-4 in combination with LangChain and semantic search to generate questions that are not only grammatically correct but also contextually aligned with the subject matter. When a student selects a subject module (e.g., Programming, English, Software Engineering, or Mathematics), the system generates questions dynamically using pre-indexed academic materials. These materials are embedded in a vector database (Pinecone) and retrieved based on semantic similarity. This ensures that the questions are domain-specific and relevant to the syllabus. Student proficiency levels are assessed continuously, enabling the system to refine the content difficulty in real time. Through this architecture, the system provides a responsive and context-aware question generation mechanism that evolves based on individual learning needs.

To support personalized learning, the system integrates a layered feedback and assessment loop. When a quiz is submitted, student performance is immediately analyzed. If the average score for a module is below 50%, the system assigns that student an "Easy" difficulty level for subsequent quizzes. A score between 50% and 75% triggers "Medium," and above 75% assigns "Hard." This difficulty classification is maintained per module and recalculated with each attempt. Redis is used to manage session states and progression history in real time. As a result, each student receives a personalized learning journey, with questions that match their current understanding and promote optimal challenge levels without overwhelming them.

The system uses Retrieval-Augmented Generation (RAG) to improve the quality and relevance of generated questions. When a student initiates a quiz, GPT-4 is augmented with domain-specific information retrieved from a vector store that contains lecture notes, slides, and official SLIIT module documents. After each quiz session, the system analyzes which questions the student got incorrect and identifies the underlying topics. In the next quiz session, the system prioritizes these topics to reinforce weak areas. Object-level metadata like topic, difficulty, and correctness history is maintained using structured JSON, which allows precise alignment with learning outcomes. This architecture ensures that questions are not only syntactically valid but also pedagogically sound and context-sensitive.

Research Objectives

AI-Based Question Generation for Software Engineering

The first objective is to design a question generation system tailored to Software Engineering concepts such as OOP, SDLC, and algorithms. Using GPT-4 and LangChain, the system will generate adaptive and context-specific questions that evaluate both theoretical understanding and practical application. Real-time adaptability ensures that question complexity aligns with student proficiency.

AI-Optimized Programming Question Generation

The second objective is to build a scalable system that generates personalized programming questions using RAG and sandbox environment. Questions will adapt to the student’s skill level and align with course objectives. The system ensures contextual accuracy and pedagogical soundness. The questions will be adapted to previous student mistakes.

Adaptive Quiz System for English Language Learning

The third objective is to enhance English language learning through an AI-powered system that generates quizzes for Grammar, Vocabulary, and Writing. The system will provide real-time feedback and explanations, helping students strengthen weak areas. Gamification elements will maintain user engagement and improve motivation throughout the learning process.

Generative AI for Personalized Mathematics Learning

The fourth objective is to implement a generative AI system for mathematics that dynamically creates questions based on real-time student performance. Using LangChain, LangGraph, and Redis, the system will track progress, adjust difficulty, and ensure personalized question delivery to reinforce learning in higher education settings like SLIIT.

Methodology

Figure 1. High Level Architecture of the system.

Figure 2. RAG System Diagram.

The proposed GenAI question generating system consists of 4 main components. They are;

  1. Automated Question Generation for Software Engineering Modules
  2. Automated Question Generation for Programming Modules
  3. Automated Question Generation for English Modules
  4. Automated Question Generation for Mathematics Modules

The overall system diagram of the proposed solution designed to offer a smart, AI-powered platform for dynamically generating adaptive quizzes across multiple domains such as Software Engineering, Programming, English, and Mathematics. The system is developed to assist students, lecturers, and educational administrators by improving personalized learning and formative assessment processes. As illustrated in the diagram, users (students) can select a subject module and initiate the quiz generation process. When the request is submitted, the backend system—hosted on a FastAPI server—is triggered. This backend is integrated with OpenAI’s GPT-4 via LangChain for orchestrating dynamic question generation. To ensure contextual alignment with the curriculum, domain-specific documents (e.g., SLIIT lecture notes and textbooks) are stored and indexed in a Pinecone vector database. These documents are queried using Retrieval-Augmented Generation (RAG) to guide the model in producing relevant, high-quality questions.

Initially, each user is presented with a set of randomly generated questions from the selected module. Upon completion, the system evaluates the responses in real time. Based on the performance (e.g., <50% = Easy, 50–75% = Medium, >75% = Hard), the system classifies the user’s proficiency level for that particular module. Redis is used to maintain session-level data, including quiz history and question-wise performance metrics. If a student answers questions incorrectly, the system logs the associated topics and ensures that subsequent quizzes reinforce those weak areas. Real-time feedback is provided with explanations, and the adaptive algorithm adjusts difficulty levels accordingly. Additionally, the frontend—built with a modern and engaging UI—ensures seamless student interaction and supports responsive design across devices.

By continuously analyzing performance and tailoring content, the system not only enhances learner outcomes but also contributes to intelligent, scalable education solutions suitable for university-level deployment.
